Python developers are used to see Exceptions flying around, it's a great way to prevent application from damaging data and making serious mistakes.

When developing new applications or libraries we should think carefully about what kind of exception it could raise and especially which exceptions could end up in user hads.
Django Forms is great example of exceptions use: Forms validation is built around `ValidationError` class, however it's barely impossible for end application to get `ValidationError` in own context.
